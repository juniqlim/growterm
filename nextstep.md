# Next Step: 에이전트 코딩 최적화 터미널

## 방향
growTerm을 단순 터미널이 아닌, **에이전트 코딩에 최적화된 터미널**로 발전시킨다.

## 기능 목록

### 1. 뽀모도로 타이머
- 시작 전 이번 뽀모도로 목표 입력
- 터미널에 첫 타이핑 시 25분 타이머 자동 시작
- 타이머 종료 시 터미널 시각적 변화 (색상/오버레이 등)
- 3분간 PTY 입력 차단 → 강제 회고 시간
- 3분 후 자동으로 다음 뽀모도로 시작

### 2. AI 코칭
뽀모도로 종료 시 해당 세션의 사용 패턴과 결과를 AI에 질의하여 피드백을 제공한다.
- 25분간의 터미널 사용 패턴 수집 (명령어, 에러, 컨텍스트 클리어 횟수 등)
- AI에 분석 요청 (print 모드 등 비대화형으로)
- 3분 휴식 시간에 코칭 피드백 표시
- 목표 대비 달성도 피드백
- 예: "목표: 로그인 API 구현 → 컨텍스트 클리어 2번, 같은 에러 반복. 방향을 먼저 정리해보는 건 어떨까?"

### 3. 자동 핸드오프
사용자가 `/cle`를 입력하면 growTerm이 가로채서 자동으로 핸드오프를 수행한다.

1. 사용자가 `/cle` 입력
2. growTerm이 가로챔 (PTY로 전송하지 않음)
3. "앞으로 할 거 정리해줘" 를 자동으로 에이전트에 전송
4. 에이전트 응답을 캡처하여 저장
5. `/clear` 전송
6. 클리어 후 저장해둔 핸드오프 내용을 자동 붙여넣기

### 4. 컨텍스트 사용량 표시
터미널 상단/하단 바에 컨텍스트 잔여량을 프로그레스 바로 보여준다.
- `/usage` 응답을 파싱하여 표시
- 미해결: 사용자에게 안 보이게 자동 호출하는 방법

### 5. 응답 속도 측정/표시
- 질문 전송 → 첫 응답까지 시간 (TTFB)
- 질문 전송 → 전체 완료까지 시간
- 세션 평균 응답 속도 표기
- PTY 입출력 타이밍으로 측정

---

## 기술 포인트
- PTY 입출력 중간에서 가로채기 (growTerm이 이미 PTY 제어 가능)
- 에이전트 응답 완료 시점 감지
- 핸드오프 내용 임시 저장 및 자동 입력
